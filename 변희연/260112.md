## 📝 Jira  요약
팀 프로젝트 협업 툴인 지라(Jira)를 오늘 처음 써봤는데 생각보다 구조가 체계적이다.
앞으로 git이랑 같이 연동해서 쓰려면 기본 개념은 확실히 잡고 가야 할 듯.

### 1. 지라에서 말하는 **'이슈(Issue)'**란?
지라에서는 모든 작업 단위를 '이슈'라고 부른다. 단순히 에러만 말하는 게 아니라 내가 할 일 전부가 이슈다.

**Epic**: 큰 줄기의 기능 (ex. 로그인 시스템 구축)

**Story**: 유저 입장에서 필요한 기능 (ex. 유저는 카카오로 로그인할 수 있다)

**Task**: 실제로 내가 할 일 (ex. API 연동, UI 작업)

**Bug**: 고쳐야 할 버그

### 2. 보드(Board) 활용법
내 작업 상태를 팀원들한테 보여주는 게시판 같은 거다. 보통 아래 흐름대로 카드를 옮기면서 진행한다.

**Backlog**: 나중에 할 일 쌓아두는 곳

**To Do**: 이제 진짜 시작할 일

**In Progress**: 지금 내가 잡고 있는 작업 (제일 중요!)

**Done**: 끝!

### 3. 협업할 때 주의할 점
**담당자(Assignee) 지정**: 이 일을 누가 하는지 명확하게 하기.

**상태 업데이트**: 작업 시작하면 바로 'In Progress'로 옮기기.

**댓글 활용**: 진행 상황이나 히스토리는 댓글로 남겨야 나중에 나도 안 까먹음.

#### **한 줄 소감**
협업할 때 "그거 어디까지 됐어요?"라는 말 안 나오게 하려고 쓰는 툴인 것 같다. 익숙해지면 엄청 편할 듯!
