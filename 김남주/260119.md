# 📝 오늘 학습 내용 정리

## 1. 멘토링: 강시몬 컨설턴트님 (ERD/DB 설계)

데이터 보존의 원칙: ERD 작성 시, 유저가 입력한 데이터는 누락 없이 모두 저장하는 것을 원칙으로 합니다.

저장 방식: 데이터를 JSON 형식의 통으로 저장하는 것을 지양하고, 개별 Value(컬럼)로 분리하여 저장하는 것이 좋습니다. (정규화 및 쿼리 효율성 고려)

## 2. 프로젝트: 요구사항 정의

현황: 초기 구상한 기능의 범위가 생각보다 방대함.

액션 플랜: 불필요한 기능을 덜어내고, 핵심 기능(Core Features) 위주로 우선순위를 재설정하여 집중할 것.

## 3. 취업 특강 (커리어 전략)

대기업 진입 전략: 반드시 첫 직장이 대기업일 필요는 없습니다. 협력업체 등에서 실무 경험을 쌓은 뒤 이직하는 것도 유효한 전략입니다.

중고신입 트렌드: 최근 신입으로 지원하는 경력직(중고신입)의 평균 경력 기간은 약 1.3년 정도입니다.
