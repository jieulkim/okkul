#### Git 브랜치 전략 공유
Git flow - [ branch 전략 ]
" -f , force 금지 "
master(main) : 사용자에게 배포
release : 배포 전 최종 버그 수정 및 점검
develop : 다음 버전 출시를 위해 개발 진행
feature : 새로운 기능을 개발
hotfix : 배포 중인 서비스(master)에서 급한 버그 수정

SSAFY git lab에서는 1개의 repository만 활용 가능.
예시 - master -> develop -> 분야 별 feature 나누기 (ex. frontend-feature/sign-in , backend-feature/login , ai-feat/ML) 

release branch와 hotfix branch는 프로젝트 규모에 따라 사용.
realease - 다음 버전 개발과 현재 버전 배포 준비를 동시에 할 때
hotfix - 프로젝트가 배포 중일 때

알아보면 좋은 것
MR 템플릿
Commit Message 템플릿
branch / commit / MR 네이밍 컨벤션
